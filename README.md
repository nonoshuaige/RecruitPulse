## 1. 简历导入

导入模块是候选人数据的入口。HR 录入候选人姓名和手机号后，可导入与候选人的招聘聊天记录（JSON 格式），系统内置的 **AI 智能解析引擎** 会自动从聊天文本中提取结构化信息：性别、年龄、邮箱、学历、工作经历、项目经历等。解析结果支持人工编辑校对，并可推送候选人确认。

### 主要功能
- **新增候选人**：姓名 + 手机号录入，自动校验格式与去重
- **导入聊天记录**：支持标准 JSON 文件，校验 role/content/type/time 字段
- **AI 智能解析**：基于正则 + 关键词匹配，从聊天记录中提取教育/工作/项目信息
- **结构化编辑**：动态表单支持增删学历、工作、项目条目
- **推送确认**：将解析结果推送给候选人复核

## 2. 简历筛选

筛选模块对已解析的候选人进行智能匹配打分。HR 用自然语言输入筛选条件（如"有电商高并发经验的 Java 后端，大厂背景"），系统模拟 AI 辅助从三个维度打分排序。

### 三维评分模型
| 维度 | 权重 | 说明 |
|------|------|------|
| 硬性匹配 | 35% | 学历、大厂经历、工作年限等硬指标 |
| 经验匹配 | 45% | 输入关键词在结构化信息中的命中率 |
| 稳定性 | 20% | 基于工作时段推导，2年以上加分 |

### 操作流程
1. 输入筛选条件 → 点击"开始智能筛选"
2. 候选人按综合评分降序排列
3. 点击卡片查看结构化详情、原始聊天记录
4. **通过** → 进入面试流程池 / **淘汰** → 填写理由归档

## 3. 面试流程

流程模块管理从筛选通过到最终录用的完整面试链路，分为四个阶段：

```
待预约 ──> 已预约 ──> 待评价 ──> 已完成（通过/不通过）
```

### 面试官侧
- **空闲时间录入**：选择未来 7 天日期 + 每天 6 个时段（上午/下午各 3 个）
- **面试评价**：面试完成后填写评价，选择通过或不通过

### 候选人侧
- **选时间**：在待预约列表中，从面试官空闲时段中选择合适时间
- **接收结果**：通过/不通过 + 面试评价反馈

### 四阶段 Tab
| Tab | 说明 | 展示内容 |
|-----|------|---------|
| 待预约 | 已通过筛选，等待选时间 | 可选时段列表 |
| 已预约 | 已选好面试时间 | 面试时间 + "等待面试" |
| 待评价 | 面试已完成，等待评价 | 面试时间 + 评价表单 |
| 已完成 | 已评价归档 | 通过/不通过 + 评价摘要 |

## 4. 数据可视化

数据看板提供招聘全流程的关键指标总览，帮助 HR 和管理者一目了然地掌握招聘进度。

### KPI 数字卡片
- **待筛选**：尚未应用筛选条件的候选人
- **待预约**：已通过筛选，等待安排面试
- **已通过**：面试通过的候选人总数

### ECharts 饼图
- 候选人状态分布可视化：待筛选 / 待预约 / 已通过 / 已淘汰 / 面试不通过
- 交互式图例，支持点击切换显示
- 仅显示人数 > 0 的分类，自动过滤空数据

## 快速启动

```bash
# 安装依赖
npm install

# 启动开发服务
npm run dev
# → http://localhost:5173

# 构建生产版本
npm run build
```

项目为纯前端应用，无需后端服务。所有数据存储在浏览器 localStorage 中，首次启动自动加载 5 条示例候选人数据覆盖完整流程。
